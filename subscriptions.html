Subscriptions

We use subscriptions to allow multiple views to subscribe to a controller actions. By default the view that begins a request is subscribed to the called action, but this can be overriden by setting 

subscribe: false

in the options object that is passed to a listener definition.

To subscribe more views to a particular controller action, call the subscribe_to method on an already initialized view

      my_widget = $v.MyWidget.initialize($("#my_widget_node"))
      my_widget.subscribe_to('Controller.action_name')

By default, the update is called on the subscribed view when the action publishes. If you want to use another view function, pass its name as the second argument to the subscribe_to function:

      my_widget.subscribe_to('Controller.action_name', 'view_function')
